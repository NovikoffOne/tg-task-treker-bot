# Архитектура AI-агентов

> **Дата создания:** 2025-01-28  
> **Обновлено:** 2025-01-28

Этот раздел содержит документацию по архитектуре системы AI-агентов.

## Обзор

Система состоит из 5 специализированных AI-агентов на базе io.net AI API для автоматизации работы с задачами и проектами в Telegram боте.

## Компоненты системы

### 1. Agent Coordinator
Координирует работу всех агентов, анализирует запросы пользователя и направляет их к нужным агентам.

### 2. Orchestrator Agent
Анализирует запросы пользователя и определяет, какие действия нужно выполнить.

### 3. Task Manager Agent (ATM)
Управление задачами и проектами - создание, обновление, удаление.

### 4. Data Manager Agent (ADM)
Работа с базой данных - выполнение операций через репозитории.

### 5. Control Manager Agent (ACM)
Проверка корректности данных перед выполнением операций.

### 6. Analyze Manager Agent (AAM)
Аналитика и отчеты - статистика, анализ данных.

## Использование

### В Telegram боте
Просто напишите естественный запрос:
```
Создай новый проект id+ Polaroid Photo
Добавь тз https://docs.google.com/... к проекту 5005
Закрой задачу на доске Подготовка
```

### Программно
```python
from agents.agent_coordinator import AgentCoordinator
from database import Database

coordinator = AgentCoordinator(db=Database())
result = coordinator.process_user_message(
    user_message="Создай проект id+ Test",
    workspace_id=1,
    user_id=123456
)
```

## Модель по умолчанию

Используется **DeepSeek-R1-0528** - экономичная модель (~7B параметров) с отличным пониманием JSON и структурированных данных.
