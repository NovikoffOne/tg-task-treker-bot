- Используйте существующие паттерны кода
- Пишите тесты для всех новых функций
- Документируйте сложные решения

### Процесс разработки
1. Изучите [архитектуру системы](../architecture/system-overview.md)
2. Следуйте [плану реализации](implementation-plan.md)
3. Пишите тесты перед реализацией (TDD)
4. Проверяйте код через линтеры
5. Обновляйте документацию

## Тестирование

### Написание тестов
- Пишите unit тесты для всех компонентов
- Создавайте интеграционные тесты для сценариев
- Используйте [руководство по тестированию](../testing/README.md)

### Запуск тестов
```bash
# Все тесты
pytest task_tracker_bot/tests/ -v

# Конкретный файл
pytest task_tracker_bot/tests/test_repositories.py -v

# С покрытием
pytest task_tracker_bot/tests/ --cov=task_tracker_bot --cov-report=html
```

## Структура проекта

```
task_tracker_bot/
├── models/          # Модели данных
├── repositories/    # Репозитории (Data Access)
├── services/        # Бизнес-логика
├── handlers/        # Обработчики команд
├── callbacks/       # Обработка callback-запросов
├── utils/           # Утилиты
├── agents/          # AI-агенты
└── tests/           # Тесты
```
