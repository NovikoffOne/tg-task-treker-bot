# Тест-кейсы для ручного тестирования Telegram Task Tracker Bot

> **Дата создания:** 2025-01-28  
> **Версия:** 2.0

## Структура тест-кейсов

Каждый тест-кейс содержит:
- **ID** - уникальный идентификатор теста
- **Название** - краткое описание теста
- **Приоритет** - High/Medium/Low
- **Предусловия** - что должно быть выполнено перед тестом
- **Шаги выполнения** - пошаговые действия
- **Ожидаемый результат** - что должно произойти
- **Статус** - Pass/Fail/Blocked

## 5. Управление задачами

### TC-401: Создание задачи (интерактивный режим)

**Приоритет:** High  
**Предусловия:** Существует хотя бы одна доска с колонками

**Шаги выполнения:**
1. Отправить команду `/newtask`
2. Выбрать доску (ввести номер)
3. Выбрать колонку (ввести номер)
4. Ввести название задачи
5. Ввести описание задачи (или пропустить)

**Ожидаемый результат:**
- Задача создана успешно
- Бот подтверждает создание
- Задача отображается на доске в выбранной колонке
- Задаче присвоен ID

**Статус:** ⬜

---

### TC-402: Просмотр задачи

**Приоритет:** High  
**Предусловия:** Существует задача с известным ID

**Шаги выполнения:**
1. Отправить команду `/task 123` (где 123 - ID задачи)
2. Альтернативно: открыть доску через `/board <название>` и кликнуть на задачу (если доступно)

**Ожидаемый результат:**
- Отображается полная информация о задаче
- Показаны: название, описание, доска, колонка, приоритет
- Отображены метки и поля задачи
- Показаны даты создания и обновления
- Показаны даты started_at, completed_at, deadline (если установлены)
- Показан ответственный (если назначен)
- Можно просмотреть задачу как через команду, так и кликом с доски

**Статус:** ⬜

- Задача отображается в новой колонке на доске

**Статус:** ⬜
