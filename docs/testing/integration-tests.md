# Интеграционные тесты

> **Дата создания:** 2025-01-28  
> **Обновлено:** 2025-01-28

Руководство по интеграционному тестированию Task Tracker Bot.

## Обзор

Интеграционные тесты проверяют взаимодействие между компонентами системы и полные сценарии использования.

## Запуск интеграционных тестов

```bash
# Все интеграционные тесты
pytest task_tracker_bot/tests/test_integration.py -v

# Конкретный тест
pytest task_tracker_bot/tests/test_integration.py::test_full_project_workflow -v
```

## Типы интеграционных тестов

### 1. Полный workflow создания проекта

Проверяет полный цикл создания проекта:
1. Создание пространства
2. Создание досок с колонками
3. Создание проекта
4. Автоматическое создание задач на досках
5. Проверка связей между задачами и проектом

### 2. Синхронизация полей в реальном сценарии

Проверяет синхронизацию полей между задачами проекта:
1. Создание проекта с задачами
2. Добавление поля к одной задаче проекта
3. Проверка, что поле добавлено ко всем задачам проекта
4. Изменение поля
5. Проверка обновления во всех задачах

### 3. Перемещение задач между колонками

Проверяет перемещение задач:
1. Создание задачи в колонке "Очередь"
2. Перемещение в колонку "В работе"
3. Проверка обновления позиции
4. Проверка обновления дашборда проекта (если задача проекта)

### 4. Обновление этапов дашборда

Проверяет автоматическое определение этапа проекта:
1. Создание проекта с задачами на разных досках
2. Перемещение задачи "Дизайн" в колонку "Готово"
3. Проверка обновления этапа дашборда на "design"
4. Перемещение задачи "Разработка" в колонку "В работе"
5. Проверка обновления этапа на "development"
